Project Overview

This project demonstrates the design and implementation of IoT-enabled sensor systems using Arduino, ESP-32, and various sensors for real-time environmental monitoring. Sensor data is collected, processed by the microcontroller, and transmitted to the ThingSpeak cloud platform for live visualization, analysis, and alerting.

The system monitors temperature and humidity using a DHT11 sensor, with data uploaded to ThingSpeak Channel #3060088 at regular intervals. Additional experiments involved ultrasonic distance sensors, infrared (IR) sensors, and Raspberry Pi integration for edge processing.

System Architecture

The IoT system follows a three-tier architecture:

1. Sensor Layer (Edge Devices)

  • DHT11 — Digital temperature and humidity sensor. Measures temperature (0–50°C, ±2°C accuracy) and relative humidity (20–80%, ±5% accuracy).
  • HC-SR04 Ultrasonic Sensor — Measures distance (2–400 cm) using time-of-flight of ultrasonic pulses. Used for proximity detection and level monitoring.
  • IR Sensor — Infrared obstacle detection for short-range proximity sensing and object counting.

2. Processing Layer (Microcontrollers)

  • Arduino Uno — Primary microcontroller for sensor interfacing, ADC conversion, and serial communication.
  • ESP-32 — Wi-Fi-enabled microcontroller that reads sensor data and transmits it to ThingSpeak via HTTP API over Wi-Fi.
  • Raspberry Pi — Used for edge computing tasks, data logging, and local visualization.

3. Cloud Layer (ThingSpeak)

  • Data uploaded via ThingSpeak REST API every 15 seconds
  • Two fields: Field 1 (Humidity %), Field 2 (Temperature °C)
  • Built-in MATLAB visualization for trend analysis
  • Public channel for remote monitoring: Channel #3060088

Code Overview

The ESP-32 firmware follows this flow:

  1. Initialize Wi-Fi connection and connect to local network
  2. Initialize DHT11 sensor on a digital GPIO pin
  3. In the main loop: read temperature and humidity values
  4. Validate readings (discard NaN or out-of-range values)
  5. Construct HTTP GET request with API key and field values
  6. Send data to ThingSpeak API endpoint
  7. Wait 15 seconds (ThingSpeak free tier rate limit) and repeat

Additional experiments included an ultrasonic distance sensor connected to Arduino for obstacle distance measurement, and IR sensors for object counting in a simulated conveyor setup.
